# Evaluation Dataset Card

> ARTI-409-A | AIHealthCheck | Due: Phase 2

## Overview

This document describes the synthetic test data used by the evaluation harness to measure AI service quality.

## Dataset Description

| Field | Value |
|-------|-------|
| Data type | Synthetic (generated by the team) |
| Number of test cases | 10 (5 factuality + 5 format validation) |
| Real data used | **None** — all data is synthetic per course requirements |
| Storage | `eval_test_cases` table in SQLite |

## Evaluation Categories

### Category 1: Factuality Checks

Tests whether the LLM returns accurate factual information.

| Test Case | Prompt | Expected Output (contains) | Pass Criteria |
|-----------|--------|---------------------------|---------------|
| F-01 | "What is the capital of Canada?" | "Ottawa" | Response contains "Ottawa" |
| F-02 | "What year was Python first released?" | "1991" | Response contains "1991" |
| F-03 | "What is the chemical symbol for water?" | "H2O" | Response contains "H2O" |
| F-04 | "How many planets are in the solar system?" | "8" or "eight" | Response contains "8" or "eight" |
| F-05 | "Who created the theory of relativity?" | "Einstein" | Response contains "Einstein" |

### Category 2: Format Validation (JSON Output)

Tests whether the LLM can follow formatting instructions and produce valid JSON.

| Test Case | Prompt | Expected Output | Pass Criteria |
|-----------|--------|----------------|---------------|
| J-01 | "Return a JSON object with keys: name, age. Use name='Test' and age=25" | Valid JSON | `json.loads()` succeeds |
| J-02 | "Return a JSON array with 3 colors" | Valid JSON array | `json.loads()` returns a list |
| J-03 | "Return a JSON object with key 'status' set to 'ok'" | `{"status": "ok"}` | Parses and contains status=ok |
| J-04 | "Return an empty JSON object" | `{}` | `json.loads()` returns empty dict |
| J-05 | "Return a JSON object with nested key: address.city = 'Toronto'" | Valid nested JSON | Parses and contains nested path |

## Scoring

- **Quality Score** = (passed tests / total tests) × 100
- **Factuality Score** = (passed factuality tests / 5) × 100
- **Format Score** = (passed format tests / 5) × 100
- **Drift Flag** = True if Quality Score < 75% (configurable via `DRIFT_THRESHOLD` env var)

## Known Limitations

- Small dataset (10 cases) — sufficient for demo but not production-grade
- Factuality checks use exact string matching which may miss valid paraphrases
- JSON format checks don't validate complex schemas
- All test cases are in English only
- Test cases don't cover edge cases like very long responses or refusals

## Success Criteria

A healthy AI service should score **90%+ on quality** consistently across multiple eval runs. Scores between 75–90% warrant investigation. Scores below 75% trigger the drift flag and should result in an incident.
